# meken-HMtest

git使いかた
1.目的と仕様をそろえる
タスクを小さく分けて、何を作るか・完了条件をチームで一致させます。

2.作業チケットを作る
Issueやタスク管理ツールで、担当・期限・優先度を明確にします。

3.ブランチを分けて実装する
mainで直接作業せず、機能ごとに feature/xxx などの作業ブランチで進めます。

こまめにコミットする
変更内容が追える単位でコミットし、メッセージは何をしたか分かる形にします。

リモートへpushしてPRを作る
作業ブランチをpushし、Pull Requestでレビュー依頼します。

コードレビューを行う
バグ、設計、可読性、命名、テスト不足を確認して指摘・修正します。

テストと動作確認
ローカルとCIでテストを通し、必要なら手動確認も行います。

mainへマージ
レビュー承認後にマージし、不要ブランチを削除します。

共有と振り返り
変更点、注意点、次の作業をチームに共有し、運用を改善します。
