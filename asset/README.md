# asset

## ディレクトリ構成

```text
asset/
├─ images/ (gitignore)
│   ├─ xxx.webp
│   └─ yyy.webp
├─ images.bak/
│   ├─ xxx.big.png
│   └─ yyy.jpg
├─ optimize.ipynb
├─ .gitignore
└─ README.md
```

## 同期時の注意

クローン/フェッチ直後は全ての画像の参照が切れています。各自スクリプトを叩いて表示用の画像を生成してください。

## 画像を追加・変更する際の注意

`images.bak/`に元画像を追加してください。

スクリプトを実行することで`images/`に中身がコピーされ、実際に表示される形式になります。

形式はpngまたはsvgを推奨します。
jpgは最適化時に劣化が重ね掛けされるのであまりお勧めしません。
現状webpには触れない設定なので、最適化済みであることを確認しない限り入れないでください。

ファイル名に`xxx.med.jpg`のようにサイズ指定子を付与することで、最適化時の上限サイズを決めることができます。指定子は次の3種類です：

- sml (小さい ; アイコン等)
- med (普通 ; 一般画像)
- big (大きい ; フル画面)

サイト上で表示される際の大きさを想定して適切に付与してください。
指定子がない場合サイズの変更はされません。
指定子は最適化時に削除され`xxx.webp`のようになります。
